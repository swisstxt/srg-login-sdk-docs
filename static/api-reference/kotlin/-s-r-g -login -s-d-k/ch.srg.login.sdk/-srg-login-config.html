SrgLoginConfig

fun SrgLoginConfig(clientId: String, redirectUri: String, appIdentity: AppIdentity, environment: Environment, postLogoutRedirectUri: String? = null, enableLogging: Boolean = false, tokenRefreshConfig: TokenRefreshConfig = TokenRefreshConfig(), enableErrorTracking: Boolean = true): SrgLoginConfig

Apple-friendly factory for SrgLoginConfig that hides ROPC-specific parameters.

SKIE's co.touchlab.skie.configuration.DefaultArgumentInterop generates Swift overloads for regular Kotlin functions but not for data class constructors. This factory function works around that limitation: SKIE treats it as an ordinary function and generates one Swift overload per optional parameter, so iOS and tvOS callers can omit any trailing parameter.

The ROPC parameters (SrgLoginConfig.enableRopcFlow, SrgLoginConfig.clientSecret) are intentionally excluded. Use withRopcFlow after construction when the Resource Owner Password Credentials grant is required (e.g. tvOS where no browser is available).

Example — minimal (required params only):

let config = SrgLoginConfig(
    clientId: "your-client-id",
    redirectUri: "app://callback",
    appIdentity: identity,
    environment: .prod
)
